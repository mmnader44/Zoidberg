# Grading Criteria

| Code | Description |
|------|-------------|
| ntb_delivery | Students deliver a functional Jupyter notebook |
| ntb_intro | Students introduce relevantly their notebook (overview, abstract,  requirements, objectives, etc.) |
| ntb_q&a | At least 3 relevant questions are clearly defined and answered |
| ntb_format | Students export their notebook in another format to ease sharing |
| ntb_clarity | The notebook contains clear explanations about the methodology, the processes and the code |
| ntb_summary | Students summarize their results, performance metrics and insights in a cross-platform format (such as PDF) |
| data_visuals | The notebook contains readable data visualizations (diagrams, charts, etc.) which are well-presented (title + legend + labels) |
| data_usage | Students split the dataset into 3 sets (training + validation + testing), with relevant proportions |
| data_variation | Students introduce some variations in some images of the dataset (e.g. luminosity, zoom factor, translation, rotation, etc.) |
| data_creation | Students batch create a lot of images using a script in order to increase the size of the original dataset |
| data_persistency | Training results are stored (in a file or assimilated) to allow re-running the binary without re-training |
| tools_data | Students use efficiently at least 3 different packages for data manipulation and representation (and they can explain it) |
| tools_ai | Students use efficiently any machine learning package for algorithmic procedures (and they can explain it) |
| tools_justify | Students can justify, in a professional way, why they choose a package rather than another one |
| proc_preprocess | Students pre-process the raw data, including at least data profiling AND data cleansing |
| proc_tvt | Students apply a train-validation-test procedure (and they can explain it) |
| proc_cv | Students apply a cross-validation procedure and compare its results with a train-test split (and they can explain it) |
| proc_speed | Students normalize the images during pre-processing in order to speed up the training process |
| proc_pipelines | Students use pipelines to organize, ensure reliability, and automate the entire flow of data processing and model training |
| algo_exploration | Students explore and test various machine learning methods and compare their results |
| algo_metrics | Students choice for a suitable algorithm is based on the appropriate evaluation metric(s) |
| algo_tuning | Students tune the parameters of any machine learning model (and they can explain it) |
| algo_reduction | Students apply a reducing dimensions technique (LDA, PCA, t-SNE, etc.) |
| algo_optimization | Students implement some optimization techniques (like hyperparameter tuning) and effective feature engineering |
| versioning_basics | Students use a versioning tool with a proper workflow, including regular and atomic commits with descriptive messages, and a gitignore file |
| presentation | The project is presented in a professional way using a relevant support (slides and/or demo) |
| argumentation | Students support their reflections and defend their choices with well-developed, illustrated arguments that highlight their analytical skills |
